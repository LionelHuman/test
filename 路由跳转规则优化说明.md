# 路由跳转规则优化说明

## 📋 修改内容总结

### 1. 路由配置调整

**文件：** `eduswap-ui/src/router/index.js`

**修改前：**
- 管理员后台首页：`/index` → `views/index.vue`
- 前台商城首页：`/` → `views/front/Home.vue`

**修改后：**
- 管理员后台首页：`/dashboard` → `views/index.vue`
- 前台商城首页：`/index` → `views/front/Home.vue`

### 2. 登录跳转逻辑调整

**文件：** `eduswap-ui/src/views/login.vue`

**修改内容：**
```javascript
// admin 角色跳转到后台管理系统（/dashboard）
// 其他角色（customer、student 等）跳转到前台商城（/index）
const isAdmin = roles && roles.includes('admin')

if (isAdmin) {
  // 管理员跳转到后台管理系统首页 /dashboard
  this.$router.push({ path: this.redirect || '/dashboard' }).catch(()=>{})
} else {
  // 学生用户等其他用户跳转到前台商城首页 /index
  const targetPath = (this.redirect && this.redirect !== '/dashboard' && this.redirect !== '/index') 
    ? this.redirect 
    : '/index'
  this.$router.push({ path: targetPath }).catch(()=>{})
}
```

### 3. 侧边栏菜单过滤调整

**文件：** `eduswap-ui/src/layout/components/Sidebar/index.vue`

**修改内容：**
- 过滤掉前台商城首页菜单（`/index` 和 `/`）
- 确保后台管理系统左侧菜单不显示前台页面入口

## 🎯 新的跳转规则

### 登录时自动跳转

| 用户类型 | 角色 | 登录后跳转路径 | 对应页面 |
|---------|------|--------------|---------|
| 管理员 | admin | `/dashboard` | 后台管理系统首页 |
| 学生用户 | customer/student | `/index` | 前台商城首页 |
| 普通用户 | common | `/index` | 前台商城首页 |

### 路由访问规则

| 访问路径 | 布局组件 | 页面组件 | 说明 |
|---------|---------|---------|------|
| `/dashboard` | Layout | views/index.vue | 后台管理系统首页（管理员专用） |
| `/index` | FrontLayout | views/front/Home.vue | 前台商城首页（学生用户专用） |
| `/product/:id` | FrontLayout | views/front/ProductDetail.vue | 商品详情页 |
| `/publish` | FrontLayout | views/front/Publish.vue | 发布商品页 |
| `/user/center` | FrontLayout | views/front/UserCenter.vue | 个人中心 |

## 🔍 关键改动点

### 1. 路由路径重新规划

```javascript
// 后台管理首页
{
  path: '/dashboard',
  component: Layout,
  children: [{
    path: 'index',
    component: () => import('@/views/index'),
    name: 'Dashboard',
    meta: { title: '后台首页' }
  }]
}

// 前台商城首页
{
  path: '/index',
  component: FrontLayout,
  children: [{
    path: '',
    component: () => import('@/views/front/Home'),
    name: 'FrontHome',
    meta: { title: '购物首页' }
  }]
}
```

### 2. 角色判断逻辑保持不变

仍然使用 `roles.includes('admin')` 来判断是否为管理员

### 3. 跳转路径调整

- 管理员：从 `/index` 改为 `/dashboard`
- 学生用户：从 `/` 改为 `/index`

## 📝 注意事项

1. **数据库菜单配置**
   - 如果后台管理系统中已有菜单配置，可能需要更新 sys_menu 表
   - 确保后台菜单不包含 `/index` 路径

2. **重定向处理**
   - 之前访问 `/index` 的书签会自动跳转到前台商城
   - 管理员需要访问 `/dashboard` 才能进入后台

3. **权限控制**
   - 后端接口权限不受影响
   - 前端路由守卫会根据角色自动过滤

4. **前台页面跳转路径统一**
   - 所有前台页面的跳转统一使用 `/index`
   - 已修复以下页面的跳转路径：
     - 发布商品成功后 → 跳转到 `/user/center`
     - 商品详情验证失败 → 跳转到 `/index`
     - 前台 Logo 点击 → 跳转到 `/index`
     - 退出登录 → 跳转到 `/index`
   - 后台标签页关闭后 → 跳转到 `/dashboard`

## 📞 问题排查

如果测试失败，请检查：

1. **查看浏览器控制台输出**
   - 是否有路由警告
   - 用户角色是否正确

2. **检查路由配置**
   - 确认 `/dashboard` 和 `/index` 的路由定义
   - 确认组件引用路径正确

3. **清除缓存**
   - 清除浏览器缓存和本地存储
   - 重启前端服务
