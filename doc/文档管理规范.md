## 📁 开发日志管理规范

### 1. 存放位置

**固定位置**：`doc/开发日志/`

**优点**：
- 📌 位置固定，易于查找
- 📂 与其他文档分离，避免混乱
- 🔝 置顶显示，优先级高

### 2. 文件命名

**格式**：`YYYY-MM-DD-开发日志.md`

**示例**：
- `2026-03-12-开发日志.md`
- `2026-03-13-开发日志.md`

**规则**：
- 使用日期排序，自动按时间顺序排列
- 简洁明了，一目了然
- 避免中文日期格式混乱

### 3. 生成时间

**触发条件**：
- 每日工作结束时
- 用户明确提出要求时
- 完成重要功能开发后

**生成方式**：
- 自动整理当日对话记录
- 归纳技术实现细节
- 总结问题和解决方案

### 4. 文档内容

**必选章节**：
1. ✅ 工作概述
2. ✅ 具体工作内容
3. ✅ 问题与解决
4. ✅ 文件清单
5. ✅ 经验总结
6. ✅ 后续计划

**可选章节**：
- 技术要点总结
- 成果展示
- 优化建议

## 🔗 文档关联规范

- 后台首页数据真实化改造

- 修复权限认证问题"

## 🎯 文档使用指南

### 查看今日日志

**Mac/Linux**：
```bash
cd doc/开发日志
open $(ls -t *.md | head -1)
```

**Windows**：
```cmd
cd doc\开发日志
start $(dir /b /o-d *.md | findstr /n "^1" | cut -d: -f2)
```

### 查看历史日志

1. 打开 `doc/开发日志/` 目录
2. 按日期查找文件
3. 使用 Markdown 阅读器打开

### 搜索内容

**Mac/Linux**：
```bash
cd doc/开发日志
grep -r "关键词" *.md
```

**Windows**：
```cmd
cd doc\开发日志
findstr /s /i "关键词" *.md
```
