# 侧边栏菜单优化

## 🎯 问题描述

### 问题现象

从你的截图可以看到，左侧边栏有两个"首页"相关的菜单项：

1. **"首页"** - 后台管理系统的 dashboard
2. **"EduSwap 二手交易首页"** - 前台购物首页

### 问题分析

- **重复**：两个"首页"容易混淆
- **冗长**："EduSwap 二手交易首页" 太长，占用侧边栏空间
- **不清晰**：没有明确区分前后台

---

## ✅ 解决方案

### 修改内容

**文件**：`eduswap-ui/src/router/index.js`

**修改前**：
```javascript
{
  path: '/',
  component: () => import('@/views/front/Home'),
  name: 'FrontHome',
  meta: { title: 'EduSwap 二手交易首页' }
}
```

**修改后**：
```javascript
{
  path: '/',
  component: () => import('@/views/front/Home'),
  name: 'FrontHome',
  meta: { title: '购物首页' }
}
```

## 🎨 命名建议

### 方案对比

| 方案 | 名称 | 优点 | 缺点 |
|------|------|------|------|
| **原方案** | EduSwap 二手交易首页 | 完整描述 | 太长、重复 |
| **方案一** | 购物首页 ✅ | 简洁、明确 | - |
| **方案二** | 前台首页 | 区分前后台 | 较技术化 |
| **方案三** | 二手市场 | 生动 | 不够直接 |

### 最终选择

**购物首页** ✅

理由：
1. 简洁（4 个字）
2. 明确是购物相关
3. 与"首页"（后台）区分开
4. 符合用户习惯

## 💡 其他优化建议

### 1. 图标优化

可以为"购物首页"添加一个购物车图标：

```javascript
meta: { 
  title: '购物首页',
  icon: 'el-icon-shopping-cart-2'
}
```

### 2. 菜单排序

建议调整菜单顺序：

```javascript
{
  path: '/',
  component: () => import('@/views/front/Home'),
  name: 'FrontHome',
  meta: { 
    title: '购物首页',
    icon: 'el-icon-shopping-cart-2'
  }
},
{
  path: '/publish',
  component: () => import('@/views/front/Publish'),
  name: 'Publish',
  meta: { 
    title: '发布商品',
    icon: 'el-icon-plus'
  }
},
```

### 3. 移除冗余菜单

如果"购物首页"和"首页"仍然让你觉得冗余，可以考虑：

**方案 A**：只保留"首页"（后台 dashboard）
- 顾客登录后默认进入购物首页（`/`）
- 但侧边栏不显示"购物首页"菜单项

**方案 B**：只保留"购物首页"
- 管理员登录后显示"购物首页"
- 通过"查看前台购物网"按钮访问

## ⚠️ 注意事项

### 1. 缓存清理

修改后需要：
- 刷新浏览器（Ctrl + F5）
- 或清除缓存后刷新

### 2. 菜单权限

确保菜单权限配置正确：
- 管理员：可以看到所有菜单
- 顾客：只看到前台相关菜单

### 3. 路由名称

- 路由 name 保持为 `FrontHome`
- 只修改 `meta.title` 显示文本
