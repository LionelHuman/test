# 登录跳转错误修复

## 🐛 问题描述

### 错误现象

1. **顾客登录时报错**：
   ```
   java.lang.NumberFormatException: For input string: ":id"
   ```

2. **错误堆栈**：
   ```
   Failed to convert value of type 'java.lang.String' to required type 'java.lang.Long'
   For input string: ":id"
   ```

3. **用户截图显示**：
   - 登录 URL：`http://localhost:1024/login?redirect=%2Findex`
   - 错误提示：密码输入错误 5 次，账户锁定 10 分钟

---

## 🔍 问题分析

### 根本原因

1. **redirect 参数问题**：
   - 顾客登录时，redirect 参数是 `/index`（后台首页）
   - 登录成功后，系统尝试跳转到 `/index`
   - 但顾客没有权限访问后台，可能触发路由守卫或权限拦截
   - 导致路由参数解析错误

2. **路由参数传递问题**：
   - 商品详情路由：`/product/:id`
   - 如果 `:id` 参数为空或无效，后端会尝试解析字符串 `":id"`
   - 导致 `NumberFormatException`

3. **权限问题**：
   - 顾客用户没有后台 API 权限（`biz:product:query`）
   - 访问后台接口会被 Spring Security 拦截

---

## ✅ 解决方案

### 修改 1：优化登录跳转逻辑

**文件**：`eduswap-ui/src/views/login.vue`

**修改内容**：
```javascript
this.$store.dispatch("Login", this.loginForm).then(() => {
  const roles = this.$store.getters.roles
  
  if (roles && roles.includes('admin')) {
    // 管理员 → 后台首页
    this.$router.push({ path: this.redirect || '/index' }).catch(()=>{})
  } else {
    // 其他用户 → 前台首页
    // 智能处理 redirect 参数
    const targetPath = (this.redirect && this.redirect !== '/index' && this.redirect !== '/') 
      ? this.redirect 
      : '/'
    this.$router.push({ path: targetPath }).catch(()=>{})
  }
}).catch(() => {
  // ...
})
```

**优化点**：
1. 过滤掉 `/index` 和 `/` 的 redirect 参数
2. 只有有效的 redirect 参数才会被使用
3. 默认跳转到前台首页 `/`

---

### 修改 2：商品详情页参数验证

**文件**：`eduswap-ui/src/views/front/ProductDetail.vue`

**修改内容**：
```javascript
created() {
  // 验证 productId 是否存在
  if (!this.productId) {
    this.$message.error('商品 ID 不能为空')
    this.$router.push('/') // 返回前台首页
    return
  }
  this.loadProduct()
}
```

**优化点**：
1. 创建时验证路由参数
2. 参数无效时提示用户并跳转首页
3. 避免向后端发送无效请求

## ⚠️ 注意事项

### 1. redirect 参数验证

**无效 redirect**：
- `/index` → 后台首页（顾客不能访问）
- `/` → 前台首页（默认）

**有效 redirect**：
- `/product/123` → 商品详情
- `/publish` → 发布商品
- `/user/center` → 个人中心

### 2. 路由参数验证

**商品详情页必须验证**：
```javascript
if (!this.productId) {
  this.$message.error('商品 ID 不能为空')
  this.$router.push('/')
  return
}
```

### 3. 权限控制

**顾客用户权限**：
- ✅ 访问前台页面（`/`, `/product/:id`, `/publish`, `/user/center`）
- ❌ 访问后台页面（`/index`, `/system/user`, 等）
- ❌ 调用后台 API（需要 `biz:product:query` 权限）
