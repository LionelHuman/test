# 登录角色跳转问题修复说明

## 问题描述

**现象：**
- 普通用户（customer 角色）登录后，跳转到了后台管理页面（`/index`）
- 应该跳转到前台购物页面（`/`）

**原因：**
- 登录页面的角色判断逻辑不够清晰
- 需要明确区分管理员角色和普通用户角色

## 修复方案

### 1. 修改登录页面逻辑

**文件：** `eduswap-ui/src/views/login.vue`

**修改内容：**
```javascript
// 获取用户角色信息
const roles = this.$store.getters.roles
console.log('用户角色:', roles) // 调试输出

// 根据角色决定跳转页面
// 只有 admin 角色才能访问后台，其他角色（customer、common 等）都跳转到前台
const isAdmin = roles && roles.includes('admin')

if (isAdmin) {
  // 管理员跳转到后台首页
  this.$router.push({ path: this.redirect || '/index' }).catch(()=>{})
} else {
  // 其他用户（顾客、普通员工）跳转到前台购物首页
  const targetPath = (this.redirect && this.redirect !== '/index' && this.redirect !== '/') 
    ? this.redirect 
    : '/'
  this.$router.push({ path: targetPath }).catch(()=>{})
}
```

**修改说明：**
- 添加了调试输出，方便查看用户角色
- 明确了角色判断逻辑：只有 `admin` 角色才能访问后台
- 其他角色（`customer`、`common` 等）一律跳转到前台

### 2. 角色配置说明

**数据库表：** `sys_role`

**角色配置：**
| role_key | role_name | 说明 | 跳转目标 |
|----------|-----------|------|----------|
| `admin` | 超级管理员 | 后台管理系统 | `/index`（后台首页） |
| `customer` | 顾客角色 | 前台购物用户 | `/`（前台首页） |
| `common` | 普通员工 | 前台购物用户 | `/`（前台首页） |

## 常见问题

### 问题 1：普通用户还是跳转到后台

**原因：** 该用户可能被分配了 `admin` 角色

**解决方法：**
```sql
-- 查询用户的角色
SELECT u.user_name, r.role_key, r.role_name
FROM sys_user u
JOIN sys_user_role ur ON u.user_id = ur.user_id
JOIN sys_role r ON ur.role_id = r.role_id
WHERE u.user_name = 'customer1';
```

**修正角色：**
```sql
-- 移除 admin 角色
DELETE FROM sys_user_role 
WHERE user_id = (SELECT user_id FROM sys_user WHERE user_name = 'customer1')
AND role_id = (SELECT role_id FROM sys_role WHERE role_key = 'admin');

-- 添加 customer 角色
INSERT INTO sys_user_role (user_id, role_id)
SELECT u.user_id, r.role_id
FROM sys_user u, sys_role r
WHERE u.user_name = 'customer1' AND r.role_key = 'customer';
```

### 问题 2：修改后不生效

**原因：** 浏览器缓存未清除

**解决方法：**
1. 清除浏览器缓存和 Cookie
2. 清除 localStorage
3. 使用无痕模式测试

**清除 localStorage 命令（在控制台执行）：**
```javascript
localStorage.clear()
sessionStorage.clear()
```

## 角色配置指南

### 在若依后台配置角色

1. **登录后台管理系统**
   - 使用 admin 账号登录

2. **创建顾客角色**（如果不存在）
   - 路径：系统管理 → 角色管理 → 新增
   - 角色名称：顾客角色
   - 角色标识：`customer`
   - 显示顺序：3
   - 状态：启用
   - 菜单权限：选择前台相关权限（如果有）

3. **创建顾客用户**
   - 路径：系统管理 → 用户管理 → 新增
   - 用户账号：`customer1`
   - 用户昵称：顾客测试
   - 归属部门：选择顾客部门
   - 角色：选择"顾客角色"
   - 密码：`123456`

4. **验证角色配置**
   ```sql
   -- 查询所有角色
   SELECT role_key, role_name, role_sort FROM sys_role;
   
   -- 查询用户的角色
   SELECT u.user_name, r.role_key, r.role_name
   FROM sys_user u
   JOIN sys_user_role ur ON u.user_id = ur.user_id
   JOIN sys_role r ON ur.role_id = r.role_id;
   ```
