# EduSwap 创建顾客用户完整指南

## 📋 方案一：通过后台管理系统创建（推荐）⭐

### 步骤 1：创建顾客角色

#### 1.1 执行 SQL 创建角色
在数据库中执行以下 SQL：
```sql
INSERT INTO sys_role (role_id, role_name, role_key, role_sort, data_scope, menu_check_strictly, dept_check_strictly, status, del_flag, create_by, create_time, update_by, update_time, remark) VALUES

```

#### 1.2 验证角色创建
```sql
SELECT * FROM sys_role WHERE role_key = 'customer';
```

---

### 步骤 2：在后台创建顾客用户

#### 2.1 进入用户管理
1. 登录若依后台管理系统（使用 admin 账号）
2. 点击左侧菜单：**系统管理** → **用户管理**
3. 点击 **"新增"** 按钮

#### 2.2 填写用户信息

**基本信息**：
- 用户名：`customer1`（用于登录）
- 昵称：`顾客测试`
- 归属部门：选择"研发部门"或"测试部门"
- 手机号：`13800138001`
- 邮箱：`customer@test.com`
- 性别：男

**账号信息**：
- 登录密码：`123456`（系统会自动加密）
- 确认密码：`123456`

**角色分配**：
- ✅ 勾选 **"顾客角色"**

**其他设置**：
- 状态：✅ 正常
- 备注：顾客测试用户

#### 2.3 保存
点击 **"确定"** 按钮保存用户

---

### 步骤 3：验证用户创建

#### 3.1 在后台查看
1. 在用户列表中应该能看到新创建的用户 `customer1`
2. 可以点击"更多" → "重置密码"来修改密码

#### 3.2 数据库验证
```sql
SELECT user_id, user_name, nick_name, phonenumber, status, create_time 
FROM sys_user 
WHERE user_name = 'customer1';
```

## 📝 常用 SQL 查询

### 查询所有顾客角色
```sql
SELECT * FROM sys_role WHERE role_key = 'customer';
```

### 查询所有顾客用户
```sql
SELECT user_id, user_name, nick_name, phonenumber, status, create_time 
FROM sys_user 
WHERE user_name LIKE 'customer%'
ORDER BY create_time DESC;
```

### 查询用户角色关联
```sql
SELECT u.user_name, u.nick_name, r.role_name, r.role_key
FROM sys_user u
LEFT JOIN sys_user_role ur ON u.user_id = ur.user_id
LEFT JOIN sys_role r ON ur.role_id = r.role_id
WHERE u.user_name = 'customer1';
```

## ⚠️ 注意事项

### 1. 密码加密
- 若依框架使用 BCrypt 加密密码
- 密码格式：`$2a$10$...`
- 建议通过后台创建用户，系统自动处理加密

### 2. 角色权限
- 顾客角色默认没有后台菜单权限
- 如果需要访问后台，需要在"角色管理"中配置权限
- 建议顾客角色只用于前台

### 3. 用户状态
- 确保用户状态为"正常"（status = '0'）
- 确保删除标志为"存在"（del_flag = '0'）

### 4. 部门选择
- 用户必须归属于某个部门
- 可以选择"研发部门"或"测试部门"

## 🔧 故障排查

### 问题 1：登录提示"用户不存在"
- 检查用户名是否正确
- 检查用户状态是否正常
- 检查用户是否被删除

### 问题 2：登录提示"密码错误"
- 确认密码是否正确
- 检查密码是否被重置
- 重新在后台重置密码

### 问题 3：登录后没有权限
- 检查用户是否分配了角色
- 检查角色状态是否正常
- 检查角色是否有对应权限
