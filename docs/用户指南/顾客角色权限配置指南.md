# 顾客角色权限配置指南

## 🎯 问题说明

当前问题：
1. ❌ 顾客用户（王小二）可以访问后台管理系统
2. ❌ "返回前台首页"按钮无法正常工作
3. ❌ 顾客用户应该只能访问前台购物页面

---

## ✅ 解决方案

### 方案一：在后台配置顾客角色权限（推荐）

#### 步骤 1：登录后台（使用 admin 账号）

1. 退出当前顾客账号
2. 用 admin 账号重新登录后台

#### 步骤 2：配置顾客角色权限

1. 进入 **系统管理** → **角色管理**
2. 找到 **"顾客角色"**，点击 **"修改"**
3. 在权限配置页面：
   - **菜单权限**：❌ 不勾选任何后台菜单
   - **数据权限**：选择"全部数据"
   - **状态**：✅ 正常

4. 点击 **"确定"** 保存

#### 步骤 3：配置前台访问权限

在后台的 **系统管理** → **菜单管理** 中，添加前台菜单：

1. 点击 **"新增"** 按钮
2. 填写菜单信息：
   ```
   菜单名称：EduSwap 前台
   菜单类型：目录
   路由地址：/
   组件路径：front/Home
   图标：shopping
   显示顺序：1
   状态：显示
   ```

3. 在"顾客角色"的权限中勾选这个菜单

### 方案三：修改权限验证逻辑

#### 修改权限配置文件

编辑 `eduswap-ui/src/permission.js`：

```javascript
import router from './router'
import store from './store'
import { Message } from 'element-ui'
import NProgress from 'nprogress'
import 'nprogress/nprogress.css'
import { getToken } from '@/utils/auth'

NProgress.configure({ showSpinner: false })

// 白名单路由（不需要权限）
const whiteList = ['/login', '/register', '/404', '/401', '/']

// 前台路由（顾客可以访问）
const frontRoutes = ['/', '/product/:id', '/publish', '/user/center']

router.beforeEach(async (to, from, next) => {
  NProgress.start()

  // 设置页面标题
  document.title = to.meta.title || 'EduSwap'

  // 判断是否登录
  const hasToken = getToken()

  if (hasToken) {
    if (to.path === '/login') {
      // 已登录，跳转到首页
      next({ path: '/' })
      NProgress.done()
    } else {
      try {
        // 获取用户信息
        const { roles } = await store.dispatch('user/getInfo')
        
        // 判断是否是顾客角色
        const isCustomer = roles.includes('customer')
        
        // 如果是顾客角色，只能访问前台页面
        if (isCustomer) {
          // 检查是否在白名单中
          const isInWhiteList = whiteList.includes(to.path)
          const isFrontRoute = frontRoutes.includes(to.path)
          
          if (isInWhiteList || isFrontRoute) {
            next()
          } else {
            // 顾客不能访问后台，重定向到前台
            next('/')
            NProgress.done()
          }
        } else {
          // 管理员可以访问所有页面
          next()
        }
      } catch (error) {
        // token 失效，退出登录
        await store.dispatch('user/resetToken')
        next(`/login?redirect=${to.path}`)
        NProgress.done()
      }
    }
  } else {
    // 未登录
    const isInWhiteList = whiteList.includes(to.path)
    if (isInWhiteList) {
      next()
    } else {
      // 其他页面需要登录
      next(`/login?redirect=${to.path}`)
      NProgress.done()
    }
  }
})

router.afterEach(() => {
  NProgress.done()
})
```

## 🎯 最佳实践建议

### 1. 角色权限分离

| 角色 | 可访问范围 | 权限说明 |
|------|-----------|---------|
| admin（超级管理员） | 后台 + 前台 | 所有权限 |
| common（普通员工） | 后台 | 后台管理权限 |
| customer（顾客） | 仅前台 | 只能访问前台购物 |

### 2. 路由权限配置

```javascript
// 后台路由（需要权限）
{
  path: '/system',
  component: Layout,
  redirect: '/system/user',
  meta: { 
    title: '系统管理',
    roles: ['admin', 'common']  // 只有管理员和员工可以访问
  },
  children: [...]
}

// 前台路由（公开或需要登录）
{
  path: '/',
  component: () => import('@/views/front/Home'),
  meta: { 
    title: 'EduSwap 二手交易首页',
    requiresAuth: false  // 不需要登录
  }
}
```

### 3. 权限验证

在 `permission.js` 中配置全局权限验证，确保：
- 顾客角色只能访问前台
- 未登录用户只能访问公开页面
- 已登录用户可以访问前台所有功能

## ⚠️ 注意事项

### 1. Token 管理

- 前台和后台共用同一个 Token
- 需要在后端配置不同的权限验证逻辑

### 2. 路由模式

- 前台使用 `history` 模式
- 后台也使用 `history` 模式
- 需要配置服务器支持

### 3. 跨域问题

- 前台和后台可能在不同的端口
- 需要配置 CORS 或代理

## 🔍 故障排查

### 问题 1：点击"返回前台首页"没反应

**原因**：路由跳转被权限拦截

**解决**：
```javascript
// 使用 window.location.href 直接跳转
window.location.href = '/'
```

### 问题 2：顾客仍然可以访问后台

**原因**：权限配置不完整

**解决**：
1. 在后台配置顾客角色的菜单权限
2. 修改 `permission.js` 添加权限验证

### 问题 3：前台页面显示 401 错误

**原因**：Token 验证失败

**解决**：
1. 检查后端权限配置
2. 在 `permission.js` 中添加白名单路由
