# EduSwap 项目配置说明

## 📋 配置文件说明

项目采用**配置文件模板化**的方式，确保敏感信息不会被提交到 Git 仓库。

### 文件结构

```
eduswap-admin/src/main/resources/
├── application.yml                    # 主配置文件（可以提交）
├── application-druid.yml.template     # 数据库配置模板（可以提交）
├── application-druid.yml              # 数据库配置文件（被忽略，不提交）
└── application-druid-local.yml        # 本地开发配置文件（被忽略，不提交）
```

---

## 🚀 快速开始（克隆项目后）

### 步骤 1：复制配置文件

```bash
cd eduswap-admin/src/main/resources/

# 复制数据库配置模板
cp application-druid.yml.template application-druid.yml
```

### 步骤 2：修改配置

#### 编辑 `application-druid.yml`

```yaml
spring:
    datasource:
        druid:
            master:
                url: jdbc:mysql://localhost:3306/edu_swap?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=false&serverTimezone=GMT%2B8
                username: root          # 改为您的数据库用户名
                password: your_password # 改为您的数据库密码
```

#### 编辑 `application.yml`（Redis 配置）

```yaml
spring:
  redis:
    host: localhost      # 改为本地 Redis 地址
    port: 6379
    password:            # 如果没有密码，留空
```

### 步骤 3：初始化数据库

```bash
# 1. 创建数据库
mysql -u root -p -e "CREATE DATABASE edu_swap CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. 导入表结构和数据
mysql -u root -p edu_swap < "/Users/lionelhuman/develop/graduation project/Lingma/EduSwap/sql/ry_20250522.sql"
mysql -u root -p edu_swap < "/Users/lionelhuman/develop/graduation project/Lingma/EduSwap/sql/edu_swap.sql"
mysql -u root -p edu_swap < "/Users/lionelhuman/develop/graduation project/Lingma/EduSwap/sql/数据字典初始化.sql"
```

### 步骤 4：启动项目

使用 IDEA 启动：
1. 打开项目
2. 找到 `RuoYiApplication.java`
3. 右键 → Run

或使用 Maven：
```bash
mvn clean install
mvn spring-boot:run
```

### 步骤 5：访问系统

- 后台管理：http://localhost:8080
- 账号：admin
- 密码：admin123

## 📝 配置项说明

### 必须修改的配置

| 配置文件 | 配置项 | 说明 |
|---------|--------|------|
| application-druid.yml | datasource.druid.master.username | 数据库用户名 |
| application-druid.yml | datasource.druid.master.password | 数据库密码 |

| 配置文件 | 配置项 | 说明 | 默认值 |
|---------|--------|------|--------|

| application-druid.yml | datasource.druid.statViewServlet.login-password | Druid 控制台密码 | admin123 |

## 🌍 环境切换

### 开发环境

使用默认配置（localhost）：

```bash
# application.yml 中激活默认配置
spring:
  profiles:
    active: druid
```

### 生产环境

创建 `application-prod.yml`：

```bash
cp application-druid.yml.template application-prod.yml
```

修改配置后，在 `application.yml` 中激活：

```yaml
spring:
  profiles:
    active: prod
```

## ❓ 常见问题

### 1. 数据库连接失败

**错误**：`Communications link failure`

**解决**：
- 检查 MySQL 服务是否启动
- 检查数据库地址、端口是否正确
- 检查防火墙设置

### 2. Redis 连接失败

**错误**：`Cannot get Jedis connection`

**解决**：
- 检查 Redis 服务是否启动
- 检查 Redis 密码是否正确

### 3. 端口被占用

**错误**：`Port 8080 is already in use`

**解决**：
- 修改 `application.yml` 中的 `server.port`
- 或关闭占用 8080 端口的程序

## 🎯 最佳实践

1. **使用配置文件模板**
   - 提交 `.template` 文件
   - 忽略实际配置文件

2. **本地开发使用独立配置**

   - 使用 localhost 配置

3. **生产环境配置隔离**

   - 通过环境变量注入敏感信息

   - Token 密钥定期更换
   - 数据库密码定期更换
