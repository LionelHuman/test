## ✅ 解决方案

### 方案一：重新登录（推荐）

**步骤**：
1. 退出当前登录
2. 清除浏览器缓存
3. 重新登录系统
4. 访问首页

**原因**：
- 新添加的权限需要重新登录才能生效
- 权限信息缓存在 token 中

---

### 方案二：检查并添加角色权限

#### 1. 检查当前用户角色

```sql
-- 查看 admin 用户的角色
SELECT u.username, u.user_id, r.role_id, r.role_name, r.role_key
FROM sys_user u
LEFT JOIN sys_user_role ur ON u.user_id = ur.user_id
LEFT JOIN sys_role r ON ur.role_id = r.role_id
WHERE u.username = 'admin';
```

#### 2. 检查角色是否有权限

```sql
-- 查看 admin 用户的权限
SELECT u.username, r.role_name, m.menu_name, m.perms
FROM sys_user u
JOIN sys_user_role ur ON u.user_id = ur.user_id
JOIN sys_role r ON ur.role_id = r.role_id
JOIN sys_role_menu rm ON r.role_id = rm.role_id
JOIN sys_menu m ON rm.menu_id = m.menu_id
WHERE u.username = 'admin' AND m.menu_name = '后台首页';
```

**预期结果**：应该返回一条记录
```
username | role_name | menu_name | perms
---------|-----------|-----------|------------------
admin    | 管理员    | 后台首页  | system:home:query
```

#### 3. 如果没有权限，手动添加

```sql
-- 1. 获取后台首页菜单 ID
SELECT menu_id FROM sys_menu WHERE menu_name = '后台首页';
-- 假设返回 2067

-- 2. 获取管理员角色 ID
SELECT role_id FROM sys_role WHERE role_key = 'admin';
-- 假设返回 1

-- 3. 添加角色菜单关联
INSERT INTO sys_role_menu (role_id, menu_id) 
VALUES (1, 2067)
ON DUPLICATE KEY UPDATE role_id = role_id;
```

## 📝 常见问题

### Q1: SQL 执行了但权限不生效？

**A**: 需要重新登录让权限生效
- 退出登录
- 清除浏览器缓存
- 重新登录

### Q2: 如何确认用户是否有权限？

**A**: 执行以下 SQL
```sql
SELECT u.username, r.role_name, m.menu_name, m.perms
FROM sys_user u
JOIN sys_user_role ur ON u.user_id = ur.user_id
JOIN sys_role r ON ur.role_id = r.role_id
JOIN sys_role_menu rm ON r.role_id = rm.role_id
JOIN sys_menu m ON rm.menu_id = m.menu_id
WHERE u.username = 'admin' AND m.perms = 'system:home:query';
```

### Q3: 接口一直返回 401 怎么办？

**A**: 
1. 确认已登录系统
2. 确认 token 未过期
3. 确认用户有权限
4. 重新登录

### Q4: 前端显示"数据加载失败"但没有 401 提示？

**A**: 
- 前端的错误处理可能没有正确处理 401
- 查看浏览器 Console 确认实际错误
- 检查 Network 中的实际响应
