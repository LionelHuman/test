# 删除商品详情菜单说明

## 问题描述
后台管理系统左侧菜单中显示了"商品详情"菜单项，但该页面实际上是前台页面（使用 FrontLayout 布局），不应该在后台管理系统中显示。

## 解决方案

### 方案一：前端代码过滤（已完成）

**修改文件：** `eduswap-ui/src/layout/components/Sidebar/index.vue`

**修改内容：**
在 `filteredSidebarRouters` 计算属性中添加过滤逻辑，排除商品详情菜单：

```javascript
// 排除商品详情菜单
if (route.path === '/product/:id' || (route.meta && route.meta.title === '商品详情')) {
    return false
}
```

**效果：**
- 立即生效，无需重启后端
- 前端刷新页面即可看到效果
- 不影响数据库中的菜单数据

### 方案二：数据库删除（可选）

**SQL 脚本：** `sql/删除商品详情菜单.sql`

**执行步骤：**

1. 备份数据库（重要！）
```bash
mysqldump -u root -p edu_swap > edu_swap_backup_$(date +%Y%m%d).sql
```

2. 查询商品详情菜单
```sql
SELECT * FROM sys_menu WHERE menu_name = '商品详情' OR path LIKE '%product%detail%';
```

3. 删除菜单
```sql
-- 删除商品详情菜单
DELETE FROM sys_menu WHERE menu_name = '商品详情';

-- 删除相关子菜单和按钮权限
DELETE FROM sys_menu WHERE parent_id IN (
    SELECT menu_id FROM (
        SELECT menu_id FROM sys_menu WHERE menu_name = '商品详情'
    ) AS temp
);

-- 删除角色菜单关联
DELETE FROM sys_role_menu WHERE menu_id IN (
    SELECT menu_id FROM (
        SELECT menu_id FROM sys_menu WHERE menu_name = '商品详情'
    ) AS temp
);
```

4. 重启后端服务

**效果：**
- 永久删除菜单数据
- 需要重启后端才能生效
- 不可恢复（除非从备份还原）

## 推荐方案

**推荐使用方案一（前端过滤）**，原因如下：

1. ✅ 安全：不修改数据库，可随时回滚
2. ✅ 快速：修改后立即生效，无需重启
3. ✅ 灵活：可以在需要时轻松恢复
4. ✅ 无风险：不影响数据库结构和其他功能

## 验证方法

1. 清除浏览器缓存
2. 刷新后台管理页面
3. 检查左侧菜单，确认"商品详情"已消失
4. 测试其他菜单功能是否正常

## 注意事项

1. 商品详情页面功能仍然保留，用户可以通过前台点击商品查看
2. 删除的只是后台管理系统中的菜单入口
3. 不影响商品详情页面的正常访问和使用
